1

Windows Server 2016 Yüklənməsi və İlkin Quraşdırma

Windows Server 2016 əməliyyat sisteminin yüklənməsi və ilkin quraşdırılması prosesini addım-addım izləyəcəyik. Bu mərhələ, serverinizin əsasını təşkil edir.

1 Windows Server 2016 Quraşdırılmasının Başladılması

Windows Server 2016 quraşdırma diskini daxil edin və ya ISO faylından boot edin. Setup ekranı göründükdə quraşdırmaya başlayın.

Quraşdırma ekranı yüklənir və sistem hazırlıq vəziyyətindədir
İpucu: Quraşdırmadan əvvəl serverinizin sistem tələblərini yoxlayın. Windows Server 2016 üçün minimum 1.4 GHz prosessor, 512 MB RAM və 32 GB disk sahəsi tələb olunur.
2 Dil və Bölgə Seçimləri

Quraşdırma dilini, vaxt və valyuta formatını, klaviatura düzümünü seçin. Bu addımda serverinizin lokallaşdırma parametrlərini təyin edirsiniz.

Dil və region parametrləri seçilir
İpucu: Server administratorlarının əksəriyyəti, beynəlxalq standartlara uyğunluq üçün ingilis dilini seçir, lakin sizin mühitinizə uyğun dili seçə bilərsiniz.
3 Quraşdırma Növünün Seçilməsi

Windows Server-i yeniləmək və ya xüsusi quraşdırma arasında seçim edin. Yeni quraşdırma üçün "Custom: Install Windows only (advanced)" seçin.

Xüsusi quraşdırma növü seçilir
Diqqət: "Custom" seçimi mövcud verilənləri silə bilər. Əgər mövcud sistem üzərində quraşdırma aparırsınızsa, əvvəlcədən vacib məlumatlarınızı yedəkləyin.
4 Server Versiyasının Seçilməsi

Windows Server 2016-nın quraşdırmaq istədiyiniz versiyasını seçin. "Desktop Experience" istifadəçi interfeysi ilə tam versiyanı təklif edir, "Core" isə yalnız əmr sətri interfeysini ehtiva edir.

Desktop Experience ilə Windows Server versiyası seçilir
İpucu: "Desktop Experience" yeni başlayanlar üçün daha rahat olsa da, "Core" versiyası daha az sistem resursu istifadə edir və təhlükəsizlik baxımından daha yaxşıdır.
5 Quraşdırma Prosesi

Windows Server 2016-nın quraşdırılması prosesi başlayır. Bu mərhələdə sistem faylları köçürülür, genişləndirilir və xüsusiyyətlər quraşdırılır.

Windows Server faylları və komponentləri quraşdırılır
İpucu: Quraşdırma prosesi server avadanlığınızın gücündən asılı olaraq 20-45 dəqiqə arası çəkə bilər. Bu müddət ərzində serverə müdaxilə etməyin.
6 Sistemin Yenidən Başladılması

Quraşdırma tamamlandıqdan sonra sistem avtomatik olaraq yenidən başladılır. Bu, ilkin quraşdırmaların tətbiq edilməsi üçün vacibdir.

Windows Server quraşdırma tamamlandıqdan sonra yenidən başladılır
7 Administrator Parolunun Təyin Edilməsi

İlk giriş zamanı Administrator hesabı üçün güclü bir parol təyin edin. Bu, serverinizin təhlükəsizliyi üçün kritik əhəmiyyət daşıyır.

Administrator hesabı üçün güclü parol təyin edilir
Diqqət: Mütləq mürəkkəb və təhlükəsiz bir parol seçin (ən az 8 simvol, böyük və kiçik hərflər, rəqəmlər və xüsusi simvollar daxil olmaqla). Bu parolu təhlükəsiz bir yerdə saxlayın, çünki onu itirdikdə serveri bərpa etmək çətin ola bilər.
8 İlk Giriş

Administrator hesabınız və yeni təyin etdiyiniz parol ilə sistemə daxil olun. İlk dəfə giriş edərkən Windows Server istifadəçi profili yaradılır.

Administrator hesabı ilə ilk giriş
İpucu: İlk giriş digər girişlərə nisbətən daha çox vaxt ala bilər, çünki Windows istifadəçi profili parametrlərini konfiqurasiya edir.
9 Quraşdırma Tamamlandı

Windows Server 2016 artıq tam quraşdırılmışdır. Server Manager avtomatik olaraq açılır və sizə serverinizi idarə etmək üçün əsas interfeys təqdim edir.

Windows Server quraşdırılması tamamlanıb və Server Manager açılıb
Təbrik edirik! Windows Server 2016 uğurla quraşdırıldı. İndi növbəti addımlara keçə bilərsiniz.
2

Əsas Konfiqurasiya Ayarları

Bu bölmədə Windows Server 2016-nın əsas konfiqurasiya parametrlərini tənzimləyəcəyik. Bu addımlar serverinizin şəbəkədə düzgün işləməsi üçün vacibdir.

1 Serverin Host Adının Dəyişdirilməsi

Serverə daha mənalı və tanınan bir ad verin. Bu, şəbəkədə serverin identifikasiyası üçün vacibdir və Active Directory quraşdırmadan əvvəl edilməlidir.

Kompüter adı "ADSERVER" olaraq dəyişdirilir
İpucu: Host adını təşkilatınızın adlandırma konvensiyasına uyğun seçin. Məsələn, "DC01" və ya "ADSERVER" kimi adlar domen kontrolleri üçün uyğundur. Adda boşluqlar və xüsusi simvollar istifadə etməkdən çəkinin.
2 Windows Firewall Söndürülməsi

Test və ya daxili mühitdə işləyərkən Windows Firewall-u söndürə bilərsiniz. Bu, quraşdırma zamanı yarana biləcək bağlantı problemlərini azaldır.

Windows Firewall bütün şəbəkə profilləri üçün söndürülür
Diqqət: İstehsal mühitində firewall-u tamamilə söndürmək təhlükəsizlik riski yaradır. Əvəzinə, sadəcə lazım olan portları (Active Directory portları: 389, 445, 88, 53, 135, 3268, 3269, vb.) açın.
3 Uzaqdan İdarəetmə (RDP) Funksiyasının Aktivləşdirilməsi

Remote Desktop Protocol (RDP) funksiyasını aktivləşdirərək, serverə uzaqdan idarəetmə imkanı əldə edin. Bu, fiziki olaraq serverin yanında olmadan onun idarə edilməsini təmin edir.

Remote Desktop Connection xidməti aktivləşdirilir
İpucu: Təhlükəsizliyi artırmaq üçün uzaqdan girişi yalnız administrator qrupuna məhdudlaşdırın və mümkünsə şəbəkə səviyyəsində əlavə təhlükəsizlik tədbirləri tətbiq edin.
4 IP Ünvanı Məlumatlarına Baxış

Active Directory quraşdırmadan əvvəl mövcud şəbəkə parametrlərinə baxmaq və şəbəkə adapterlərinin düzgün quraşdırıldığını yoxlamaq vacibdir.

ipconfig əmri ilə mövcud şəbəkə məlumatlarını göstərir
İpucu: "ipconfig /all" əmri daha ətraflı məlumat əldə etməyə imkan verir. Bu əmrlə MAC ünvanı, DNS server ünvanları və digər şəbəkə konfiqurasiyalarını görə bilərsiniz.
5 Statik IP Ünvanı Təyin Edilməsi

Domen kontrolleri üçün statik IP ünvanı təyin etmək çox vacibdir. DHCP-dən alınan dinamik ünvanlar domen xidmətlərinin fəaliyyətində problemlər yarada bilər.

Şəbəkə adapterin xüsusiyyətlərində statik IP konfiqurasiyası edilir
Diqqət: Statik IP ünvanını təyin edərkən şəbəkə administratoru ilə məsləhətləşin. Təkrarlanan IP ünvanları şəbəkə konfliktinə səbəb ola bilər. DNS server kimi eyni server IP ünvanını göstərin.
6 Vaxt Qurşağının Tənzimlənməsi

Active Directory-nin düzgün işləməsi üçün vaxt qurşağını düzgün konfiqurasiya etmək vacibdir. Kerberos autentifikasiyası zamanı dəqiq vaxt tələb olunur.

Sistem vaxt qurşağı düzgün bölgə ilə sinxronlaşdırılır
İpucu: Active Directory mühitlərində bütün komputerlərin vaxtı domen kontrolleri ilə avtomatik sinxronlaşdırılır. Domen kontrollerinin vaxtı isə bir xarici NTP serverlə sinxronlaşdırıla bilər.
7 Dəyişikliklərin Tətbiqi üçün Yenidən Başladılma

Edilən dəyişikliklərin tam tətbiq olunması üçün serveri yenidən başladın. Bu, xüsusilə host adı dəyişikliyi üçün vacibdir.

Server yenidən başladılır ki dəyişikliklər tam tətbiq olunsun
İpucu: Server yenidən başladılmadan əvvəl bütün vacib əməliyyatları saxladığınızdan və ya bitirdiyinizdən əmin olun. Planlaşdırılmamış yenidən başlatmalar data itkisinə səbəb ola bilər.
3

Active Directory Domain Services (AD DS) Quraşdırılması

Bu bölmədə Windows Server 2016-da Active Directory Domain Services rolunu quraşdıracağıq. Bu, mərkəzləşdirilmiş şəbəkə idarəetməsinin əsasını təşkil edən kritik bir addımdır.

AD DS nədir? Active Directory Domain Services, şəbəkədə istifadəçi hesabları, kompüterlər, printerlar və digər resursların idarə edilməsi üçün mərkəzləşdirilmiş bir xidmətdir. Bu, təhlükəsizlik, resurs paylaşımı və şəbəkə idarəetməsi üçün əsas komponentdir.
1 Server Rolları və Xüsusiyyətlərinin Əlavə Edilməsi

Server Manager-dən "Add Roles and Features" sehrbazını başladaraq, yeni rol və xüsusiyyətlər əlavə edin. Bu, AD DS rolunu əlavə etmək üçün ilk addımdır.

Server Manager-dən "Add Roles and Features" sehrbazı başladılır
İpucu: Server rolu əlavə etmədən əvvəl, serverinizin bütün yeniləmələrinin quraşdırıldığından və sistem stabilliyinin yoxlanıldığından əmin olun.
2 Active Directory Domain Services Rolunun Seçilməsi

Server Roles bölməsində "Active Directory Domain Services" rolunu seçin. Bunu seçdikdə, tələb olunan əlavə xüsusiyyətlər avtomatik olaraq əlavə ediləcək.

Server Roles siyahısından Active Directory Domain Services seçilir
İpucu: Active Directory ilə birlikdə DNS Server rolu da quraşdırılmalıdır. Əgər avtomatik əlavə edilmirsə, əl ilə DNS Server rolunu da seçin.
3 Domen Adının Təyin Edilməsi

Yeni domen adını təyin edin. Bu, şəbəkənizdəki bütün resurslar üçün ad məkanı (namespace) yaradacaq. Adətən domain.local və ya company.com formatları istifadə olunur.

Yeni domen adı kimi "domain.local" daxil edilir
Diqqət: Domen adını diqqətlə seçin, çünki bu, qurulduqdan sonra dəyişdirilməsi çətin olan bir parametrdir. İnternet ilə əlaqəli domenlər üçün rəsmi domen adları (.com, .org, vb.) istifadə edin, daxili şəbəkələr üçün isə .local kimi qeyri-rəsmi domain adları istifadə etmək olar.
4 Directory Services Bərpa Parolunun Təyin Edilməsi

Active Directory-nin təcili bərpası üçün istifadə olunacaq parol təyin edin. Bu, domenin sağlamlığını qorumaq üçün kritik əhəmiyyət daşıyan bir təhlükəsizlik tədbiridir.

DSRM (Directory Services Restore Mode) parolu təyin edilir
Diqqət: DSRM parolunu unutmayın və təhlükəsiz bir yerdə saxlayın. Bu parol, Active Directory məlumat bazasının zədələnməsi və ya digər kritik problemlər zamanı bərpa üçün lazım olacaq.
Təbrik edirik! Active Directory Domain Services rolu uğurla quraşdırıldı və domen kontrolleri konfiqurasiyanı tamamladı. Artıq istifadəçilər, qruplar və kompüterlər yaratmağa və onları idarə etməyə başlaya bilərsiniz.
4

Active Directory İlə İşləmək

Bu bölmədə, Active Directory-nin əsas idarəetmə tapşırıqlarını və gündəlik işləri yerinə yetirməyi öyrənəcəksiniz.

İstifadəçilər və Qruplar

Active Directory Users and Computers konsolu vasitəsilə istifadəçilər, qruplar və kompüter hesablarını idarə edin. Təşkilati birimlər (OU) yaradaraq resurslarınızı məntiqi şəkildə təşkil edə bilərsiniz.

Qrup Siyasətləri

Group Policy Management konsolu ilə domen daxilindəki bütün kompüterlər və istifadəçilər üçün mərkəzləşdirilmiş qaydalar və siyasətlər tətbiq edin. Bu, şəbəkənizin təhlükəsizliyini və standartlaşdırılmasını təmin edir.

Replikasiya və Yerləşdirmə Strategiyası

Böyük təşkilatlar üçün çoxsaylı domen kontrollerləri arasında replikasiya planlaşdırması vacibdir. Sites and Services konsolu vasitəsilə replikasiya topolojiyasını idarə edin.

İpucu: Active Directory strukturunuzu planlaşdırarkən təşkilatınızın həm indiki, həm də gələcək ehtiyaclarını nəzərə alın. Yaxşı planlaşdırılmış bir AD strukturu idarəetməni asanlaşdırır və gələcək genişlənməyə imkan verir.

Active Directory İdarəetməsində İrəli Addımlar

Windows Server və Active Directory quraşdırılması yalnız başlanğıcdır. Daha dərin öyrənmə üçün aşağıdakı mövzuları araşdırın:

  • Group Policy obyektlərinin yaradılması və idarə edilməsi
  • FSMO (Flexible Single Master Operations) rollarının başa düşülməsi
  • Active Directory sağlamlığının monitorinqi və problemlərin həlli
  • Backup və Recovery strategiyaları
  • LDAP sorğuları və PowerShell ilə AD idarəetməsi

Quraşdırma Tamamlandı - Sonrakı Addımlar

Windows Server 2016 və Active Directory quraşdırması uğurla tamamlandı! İndi şəbəkənizin əsas infrastruktur komponenti hazırdır.

Serveriniz artıq domen idarəetməsi üçün hazırdır! İstifadəçilər yarada, qruplar təyin edə, kompüter hesabları əlavə edə, qrup siyasətləri ilə təhlükəsizlik qaydaları tətbiq edə və Active Directory-nin bütün üstünlüklərindən istifadə edə bilərsiniz.

İnfrastrukturunuzu daha da inkişaf etdirmək üçün aşağıdakıları nəzərdən keçirin:

  • File Server, Print Server, DHCP, vb. kimi əlavə server rollarının quraşdırılması
  • Active Directory Certificate Services (AD CS) ilə PKI infrastrukturunun qurulması
  • Exchange Server, SharePoint, və ya SQL Server kimi əlavə Microsoft xidmətlərinin inteqrasiyası
  • Azure Active Directory (AAD) ilə hibrid kimlik idarəetməsinin qurulması

Mərkəzi Autentifikasiya

Single Sign-On

İstifadəçilər şəbəkəyə bir dəfə daxil olmaqla bütün icazə verilmiş resurslara çata bilərlər. Bu, təhlükəsizliyi artırır və parolu idarə etməni asanlaşdırır.

Təhlükəsizlik Siyasətləri

Group Policy Obyektləri

Mərkəzləşdirilmiş təhlükəsizlik siyasətləri ilə bütün hesablar və cihazlar üzərində avtomatik nəzarət. Parol mürəkkəbliyi, hesab kilidləmə və digər təhlükəsizlik parametrləri.

Hierarxik Struktur

Təşkilati Vahidlər (OUs)

Obyektlərin məntiqi qruplaşdırılması şirkətin strukturunu əks etdirərək idarəetməni asanlaşdırır. Departamentlər, lokasiyalar və ya funksiyalara görə qruplaşdırma.

Active Directory Ssenariləri

Active Directory, şəbəkə resurslarını idarə etmək və təşkilati strukturları yaratmaq üçün güclü bir vasitədir. Aşağıda ən çox istifadə edilən ssenarilərdən bir neçəsi verilmişdir.

Şirkət Strukturu

Departament Əsaslı OU Təşkilatı

Təşkilati vahidləri departamentlərə görə təşkil etmək (məsələn, İnsan Resursları, İT, Satış) ən çox istifadə edilən yanaşmadır. Hər bir departament öz GPO və təhlükəsizlik siyasətlərinə malik ola bilər.

Coğrafi Yerləşim

Lokasiya Əsaslı OU Təşkilatı

Müxtəlif coğrafi yerlərdə fəaliyyət göstərən şirkətlər üçün, OU-lar ofis yerləşimlərinə görə təşkil edilə bilər (məsələn, Bakı, London, New York). Bu, yerli siyasətlərin və tənzimləmələrin tətbiq edilməsini asanlaşdırır.

Resurs Paylaşımı

Qruplar Vasitəsilə İcazə İdarəetməsi

Qruplardan istifadə edərək fayl paylaşım və tətbiq icazələrini daha effektiv şəkildə idarə etmək. Rollar əsasında qrup yerləşdirmələri ilə işçilər daxil olduqda avtomatik olaraq doğru resurslara çata bilərlər.
